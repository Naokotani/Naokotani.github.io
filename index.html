Installation

In the past I had used emacs on Arch based machines, but more recently I got a laptop with windows 11 installed and decided to keep it and try out WSL to see how it handles graphical applications. So far, it seems to work pretty well and feels like the least bad way to have access to both Linux and windows on the same computer. One minor drawback is that it is a minor pain to install Arch Linux on WSL, so I opted to just stick with Ubuntu. The issue with Ubuntu is that many of the packages in the Ubuntu Repository are fairly out of date and Emacs is no exception. Luckily this is fairly easy to do.

First I made sure some dependencies were installed with apt install build-essentials and apt-get build-dep emacs, downloaded the latest stable version from the GNU Emacs website.

Next I decompressed the tar file and once inside the uncompressed directory I ran ./configure , make and make install. I believe during that process I need to install one other package related to GTK in order to make the graphical version of Emacs work, which is certainly the preferable way to use Emacs. I kept these source files on hand for later use in order to uninstall with make uninstall.

Another issue I ran into in the past was that WSL did not have systemD in order to run an Emacs server as a service. Apparently systemd, or some approximation of it, is available on WSL, but I have not yet set it up. It is definitely on my to do list.

Package management and load modules

I use straight.el for package management so this code is necessary to install and set it up. If you want to use it with use-package blocks, you also need to add the code below.

Next I load the Elisp files in two directories. I made two directories to simply dependencies. 

;; -*- lexical-binding: t; -*-

;; Straight.el bootstrap
(defvar bootstrap-version)
(let ((bootstrap-file
      (expand-file-name "straight/repos/straight.el/bootstrap.el" user-emacs-directory))
      (bootstrap-version 5))
  (unless (file-exists-p bootstrap-file)
    (with-current-buffer
        (url-retrieve-synchronously
        "https://raw.githubusercontent.com/raxod502/straight.el/develop/install.el"
        'silent 'inhibit-cookies)
      (goto-char (point-max))
      (eval-print-last-sexp)))
  (load bootstrap-file nil 'nomessage))

;; Use straight.el for use-package expressions
(straight-use-package 'use-package)

;; Tell straight.el to use use-package expressions
(setq straight-use-package-by-default t)

;;; Load Modules
(require 'find-lisp)

;; First load modules that are required as dependencies, evil mode in this case for keybinds
(mapcar (lambda (fn)
          (load (file-name-sans-extension fn)))
        (find-lisp-find-files "~/.config/emacs/early-modules" "\\.el\\'"))

;;Then load the rest
(mapcar (lambda (fn)
          (load (file-name-sans-extension fn)))
        (find-lisp-find-files "~/.config/emacs/modules" "\\.el\\'"))

Defaults

Here I set some built in defaults to remove things like scroll bars, menus, remove the audible bell etc. I set up auto revert mode so that buffers are automatically updated to reflect changes to files from outside emacs, such as on the command line etc. I tell Emacs what modes I would like line numbers in and finally I tell emacs to put temporary files in my /tmp/ folder (This is one of the first things I do as I find it incredibly annoying have these files spread all through my directories.) 

(scroll-bar-mode -1)        ; Disable visible scrollbar
(tool-bar-mode -1)          ; Disable the toolbar
(tooltip-mode -1)           ; Disable tooltips
(set-fringe-mode 10)        ; Give some breathing room

(menu-bar-mode -1)          ; Disable the menu bar

(setq visible-bell t)       ; Disable error bell and replace with flash

(global-set-key (kbd "<escape>") 'keyboard-escape-quit) ; Binds escape to mimic default C-g keyboard-quit

(column-number-mode)        ; Gives cursor position in mode line

;; Puts Emacs customize code in seperate file. 
(setq custom-file (locate-user-emacs-file "custom-vars.el")) 
(load custom-file 'noerro 'nomessage)


(setq global-auto-revert-mode t)  ; Revert buffers when the file changes on disc
(setq global-auto-revert-non-file-buffers t) ; Reverts dired buffers when directories change on disc

;; Enable line numbers for some modes
(dolist (mode '(text-mode-hook
                prog-mode-hook
                conf-mode-hook))
  (add-hook mode (lambda () (display-line-numbers-mode 1))))

;; Override some modes which derive from the above
(dolist (mode '(org-mode-hook))
  (add-hook mode (lambda () (display-line-numbers-mode 0))))

;;Setup auto saves to save to temporary file folder, defaults to /tmp/

(setq backup-directory-alist
      `((".*" . ,temporary-file-directory)))
(setq auto-save-file-name-transforms
      `((".*" ,temporary-file-directory t)))

I rely heavily on Modus Themes for my theme because I really like how nicely it integrates into Emacs. I know some people are perhaps turned off by it's stark colors, but it is highly customizable and it is possible to use whatever colors you want. Doom Themes is another great package that has a lot more selection for color schemes.

I use Doom Mode Line because it has nice integration with all-the-icons and jives fairly well with Modus Themes.

Popper probably belongs more in a window section. It creates a toggleable drawer window for buffers like shells, scratch, message buffers or what have you. I just have it set up to display my shells as a popper window. On Linux I usually use a stand alone terminal, but on windows I find it nice to have a built in shell as I find window management on windows (ironic, I know) to be a pain in comparison to tiling window managers.

Spell Checking

Here I set up the built in spell check in text mode, which comes in especially handy in org mode documents. Since that is where I use it almost exclusively, it might be worth adding it specifically to an org-mode hook, but this setup works fine for now.

Hunspell itself is a separate command line program and has to be installed on your system where Emacs can find it to work correctly. Hunspell in the official Ubuntu repository, and it is also important to note that you may need to install support for your language separately. 

;; Enables spell checking for text mode and disable for some other modes
(dolist (hook '(text-mode-hook))
  (add-hook hook (lambda () (flyspell-mode 1))))
(dolist (hook '(change-log-mode-hook log-edit-mode-hook))
  (add-hook hook (lambda () (flyspell-mode -1))))

;; Set spell checking to use the command line program hunspell
(setq ispell-program-name "hunspell")

Completion

Which Key is a handy package that provides hints for key chords. When you hit a keybind it will give you possible follow up keys. The defaults work fine for me, but you can customize how long you want it to wait to give the hints. You can set it to take longer to open the buffer if you are fairly comfortable with your keybinds and don't want cluttering your screen unless your really need it.

Which Key Github

I use Vertico and the whole sweet of packages that work together with it. I used to use Ivy, but Vertico is integrated more closely with the built in iComplete framework and is more light weight. You can also pick and choose what other packages you want to load with it if you find that you don't need or what other that other functionality.

One issue I ran into when using strait.el was here. I need to specifically tell Emacs to load the directory with the vertico extensions because I needed them for this (define-key vertico-map (kbd "M-h") 'vertico-directory-up) which allows for much faster navigation between directories. I did not previously need to do this before setting up straight.el.

Vertico Github

In addition to Vertico I use Marginalia, Corfu, Orderless, Consult and Embark. Marginalia adds more detailed information to the completion buffer such as file permissions, file size, key binds for M-x commands and so on. Corfu adds completion at point, which is especially useful in prog-mode. Orderless allows for better fuzzy finding in the completion buffer. Consult provides "Consult provides practical commands based on the Emacs completion function completing-read." The ones I use most frequently are consult-ripgrep for searching within projects and consult-buffer which provides a better buffer switching experience. Embark similarly adds practical commands that can act upon candidates in the completion buffer such as changing file permissions or renaming files.

Org Mode

It looks like there is a lot going on here, but in reality there are just a few major things going on. Font faces and visual setup, Org Roam, capture templates, a workflow screen, projects and a few other smaller things.

I am a really big fan of Org Roam, not so much because I use the Zettelkasten (not that I think there is anything wrong with it) but because I like how it organizes my org files and makes them easy to access. I use a projects tag in my Org Roam files to narrow searches for them, but more so in order to let Org Agenda know where to look for TODOS. I also install Org Roam Dailies, which creates neatly organized notes for specific days. I like to use this as a way to take some quick notes for a meeting, and I also refile my completed TODOs there so I can see a record of when they were completed.

The customized org states are for displaying my TODOs on a custom workflow page that combines the TODOs from all of my project files. Finally, there are a few functions that allow org capture templates to be used with Org Roam. Finally, there are a few functions that allow org capture templates to be used with Org Roam.

This section more than others is heavily inspired by System Crafters Channel and aside from a few small modifications, the functions are taken from the show notes from some of the videos about Org Roam, which I highly encourage you to check out to gain a more in depth understanding of what it all does.

(defun nao/org-font-setup ()
  (require 'org-faces)
  ;; Replace list hyphen with dot
  (font-lock-add-keywords 'org-mode
                          '(("^ *\\([-]\\) "
                             (0 (prog1 () (compose-region (match-beginning 1) (match-end 1) "•"))))))

  ;; Ensure that anything that should be fixed-pitch in Org files appears that way
  (set-face-attribute 'org-block nil :foreground nil :inherit 'fixed-pitch)
  (set-face-attribute 'org-code nil   :inherit '(shadow fixed-pitch))
  (set-face-attribute 'org-level-1 nil :font "Inter" :height 200)
  (set-face-attribute 'org-level-2 nil :font "Inter" :height 180)
  (set-face-attribute 'org-level-3 nil :font "Inter" :height 160)
  (set-face-attribute 'org-level-4 nil :font "Inter" :height 140)
  (set-face-attribute 'org-level-5 nil :font "Inter" :height 140)
  (set-face-attribute 'org-level-6 nil :font "Inter" :height 140)
  (set-face-attribute 'org-block-begin-line nil :font "Fira Code retina")
  (set-face-attribute 'org-table nil  :inherit 'fixed-pitch)
  (set-face-attribute 'org-verbatim nil :inherit '(shadow fixed-pitch))
  (set-face-attribute 'org-special-keyword nil :inherit '(font-lock-comment-face fixed-pitch))
  (set-face-attribute 'org-meta-line nil :inherit '(font-lock-comment-face fixed-pitch))
  (set-face-attribute 'org-checkbox nil :inherit 'fixed-pitch))

(defun org-roam-node-insert-immediate (arg &rest args)
  (interactive "P")
  (let ((args (push arg args))
        (org-roam-capture-templates (list (append (car org-roam-capture-templates)
                                                  '(:immediate-finish t)))))
    (apply #'org-roam-node-insert args)))

(defun my/org-roam-filter-by-tag (tag-name)
  (lambda (node)
    (member tag-name (org-roam-node-tags node))))

(defun my/org-roam-list-notes-by-tag (tag-name)
  (mapcar #'org-roam-node-file
          (seq-filter
           (my/org-roam-filter-by-tag tag-name)
           (org-roam-node-list))))

(defun my/org-roam-refresh-agenda-list ()
  (interactive)
  (setq org-agenda-files (my/org-roam-list-notes-by-tag "Project")))

;; Build the agenda list the first time for the session
(my/org-roam-refresh-agenda-list)

(defun my/org-roam-project-finalize-hook ()
  "Adds the captured project file to `org-agenda-files' if the
capture was not aborted."
  ;; Remove the hook since it was added temporarily
  (remove-hook 'org-capture-after-finalize-hook #'my/org-roam-project-finalize-hook)

  ;; Add project file to the agenda list if the capture was confirmed
  (unless org-note-abort
    (with-current-buffer (org-capture-get :buffer)
      (add-to-list 'org-agenda-files (buffer-file-name)))))

(defun my/org-roam-find-project ()
  (interactive)
  ;; Add the project file to the agenda after capture is finished
  (add-hook 'org-capture-after-finalize-hook #'my/org-roam-project-finalize-hook)

  ;; Select a project file to open, creating it if necessary
  (org-roam-node-find
   nil
   nil
   (my/org-roam-filter-by-tag "Project")
   nil
   :templates
   '(("p" "project" plain "* Goals\n\n%?\n\n* Tasks\n\n** TODO Add initial tasks\n\n* Dates\n\n"
      :if-new (file+head "%<%Y%m%d%H%M%S>-${slug}.org" "#+title: ${title}\n#+category: ${title}\n#+filetags: Project")
      :unnarrowed t))))

(defun my/org-roam-capture-task ()
  (interactive)
  ;; Add the project file to the agenda after capture is finished
  (add-hook 'org-capture-after-finalize-hook #'my/org-roam-project-finalize-hook)

  ;; Capture the new task, creating the project file if necessary
  (org-roam-capture- :node (org-roam-node-read
                            nil
                            (my/org-roam-filter-by-tag "Project"))
                     :templates '(("p" "project" plain "** TODO %?"
                                   :if-new (file+head+olp "%<%Y%m%d%H%M%S>-${slug}.org"
                                                          "#+title: ${title}\n#+category: ${title}\n#+filetags: Project"
                                                          ("Tasks"))))))

;; When TODOs are changed to the complete state move the to the roam daily node for that day
(defun my/org-roam-copy-todo-to-today ()
  (interactive)
  (let ((org-refile-keep t) ;; Set this to nil to delete the original!
        (org-roam-dailies-capture-templates
          '(("t" "tasks" entry "%?"
             :if-new (file+head+olp "%<%Y-%m-%d>.org" "#+title: %<%Y-%m-%d>\n" ("Tasks")))))
        (org-after-refile-insert-hook #'save-buffer)
        today-file
        pos)
    (save-window-excursion
      (org-roam-dailies--capture (current-time) t)
      (setq today-file (buffer-file-name))
      (setq pos (point)))

    ;; Only refile if the target file is different than the current file
    (unless (equal (file-truename today-file)
                   (file-truename (buffer-file-name)))
      (org-refile nil nil (list "Tasks" today-file nil pos)))))

(add-to-list 'org-after-todo-state-change-hook
             (lambda ()
               (when (equal org-state "DONE")
                 (my/org-roam-copy-todo-to-today))))
